约束
添加约束可以在建表的时候，也可以之后加

1.主键约束primary key
唯一确定一张表中的一条记录 该字段不重复且不为空(不能为NULL，不能与之前的重复)
create table user(
id INT primary key,
name VARCHAR(20),
pwd VARCHAR(20)
)
联合主键 加起来不重复就行
create table user(
id INT,
name VARCHAR(20),
pwd VARCHAR(20),
primary key（id,name)
)

建表时忘记主键约束，之后修改
alter table user add primary key(id);
删除主键约束
alter table user drop primary key;
修改主键约束
alter table user modify id int primary key;

2.自增约束auto_increment 和主键约束一起用 自动管控id的值
create table user(
id INT primary key auto_increment,
name VARCHAR(20),
pwd VARCHAR(20)
)

3.外键约束
涉及到两个表 主表 副表
foreign key指明外键来自于哪个表
主表里面不存在的值，副表不能有
主表中被副表引用了的值，是不可以删除的

create table user(
id INT primary key,
name VARCHAR(20),
pwd VARCHAR(20)
)
create table company(
id INT primary key,
money INT,
user_id INT,
foreign key(user_id) references user(id)
)

4.唯一约束unique 不可重复
create table user(
id INT,
name VARCHAR(20) unique,
pwd VARCHAR(20)
)
create table user(
id INT,
name VARCHAR(20),
pwd VARCHAR(20),
unique(name,id,pwd) //加在一起不重复就可以
)

alter table user add unique(name);

删除唯一约束
alter table user drop index name;//这里不是unique是因为唯一约束有多个
修改唯一约束
alter table user modify name varchar(20) unique;
5.非空约束 不为null
create table user(
id INT,
name VARCHAR(20) not null,
pwd VARCHAR(20)
)
6.默认约束 default
create table user(
id INT,
name VARCHAR(20),
pwd VARCHAR(20) default ‘root’
)

数据库的设计范式 NormalForm
1.第一范式 1NF
各个属性不可再分（字段不可再分）
例如：表有地址这个属性，地址可以分为省、市、区、乡镇等，
所以整个地址设为一个字段，是不合理的。

2.第二范式 2NF
满足第一范式的基础上，要求非主属性对主属性是完全函数依赖，即不存在部分函数依赖（有主键，非主键列完全依赖于主键，不可以依赖主键的一部分）
例如：有这样一个表，它的字段有学生学号id，学生名字，课程编号id，课程名称
那么，课程名称是部分依赖主键（学生id,课程id）的，这样的表 不合理。
因为如果我没选课，这条数据就不能插入数据库。
所以需要划分为学生表和课程表。

3.第三范式 3NF
满足第二范式的基础上，要求不存在传递函数依赖
例如：有这样一个表，学生学号id，学生证id，学生证说明。
那么，学生证说明是传递依赖于学生学号，不是直接依赖。
所以需要划分为学生表和学生证表。

4.BCNF
满足第三范式的基础上，不允许主键内部之间存在依赖
例如 主键为(学生，老师) 课程
这样的关系满足第三范式，但不满足BCNF范式

范式的优缺点:
减少冗余
结构清晰
性能降低

表之间关系的解释
一对一：学生和学生证 通过外键和唯一约束

mysql事务

自动commit的开启和关闭

手动commit
begin;或者 start transaction;

事务的隔离级别
1.read uncommitted; 读未提交
2.read committed; 读已提交
3.repeatable read; 可重复读
4.serializable; 可串行化

MySQL 数据库默认的事务隔离级别是 REPEATABLE_READ (可重复读)。

修改隔离级别
设置全局的事务隔离级别
set global transaction isolation level read committed;
设置本次会话的事务隔离级别
set session transaction isolation level read committed;
